\paragraph*{Specification}
The correctness of {\sf inferWidths} is specified by the formulas: 
%
$$(tm,\,cm) \models \, {\sf inferWidths}(M) \Rightarrow ({\sf madeImplicit}(tm,M),\,cm) \models \,M.$$
$$\forall tm,cm, (tm,\,cm) \models M \wedge (tm',\,cm') \models \, {\sf inferWidths}(M) \Rightarrow tm' \preceq tm \wedge cm' = cm.$$
%
%
Here {\sf madeImplicit} is a function that makes bit-width implicit. Given a type mapping with only implicit widths components $tm$ and the original module $M$, it returns a type mapping that makes some components implicit according to the definitions in $M$.

The first formula states that if the pair of the typing mapping $tm$ and the connection mapping $ct$ conform to the program of produce by {\sf inferWidths}$(M)$, then the pair of ${\sf madeImplicit}(tm,M)$ and $ct$ must conform to the original program $M$.  

For the second formula, a typing mapping $tm' \preceq tm$ means that they involve same components and 
%regardless of implicit or not, 
the bit width of a component in $tm'$ is no greater than that in $tm$.

It states that if the pair $(tm', cm')$ conform to {\sf inferWidths} $(M)$, then for any pair of $(tm, cm)$ conform to the original program $(M)$, $tm$ and $tm'$ satisfy that $tm' \preceq tm$ and $cm = cm'$.

Since {\sf inferWidths} only modifies the widths of implicit components and has no effect on changing the connection mapping, we can focus on the goals for implicit components in $tm$. To prove the formulas through {\sf inferWidths}, we need to derive clues from the conformance and its formalization. Since {\sf inferWidths} operates on components following the topological order, we can focus on an implicit component identified as $(v,m)$ and $cgels = [e_0,\dots e_n]$ are connections connected to $(v,m)$ according to $gcm$. We can prove that:

\begin{lemma}
Any implicit components in $M$ appears in the topologically sorted  $ts$.
\end{lemma}   

\begin{lemma} \label{lemma:bitwidthmax}
The bit widths of $(v,m)$ after {\sf inferWidths} is updated to the maximum width among types of its connected expressions.
\end{lemma}  

\begin{lemma}
Every connected expression to $(v,m)$ in $M$ is stored in $gcm(v,m)$, regardless of the order of statements.
\end{lemma}
