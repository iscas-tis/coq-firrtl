Our goal is to construct a connection mapping $cm'$ from $tm$, $tmap$ and $ss$ (the sequence of statements in $M$). 
For this purpose, we define two relations $cm_1 \xrightarrow[tm, tmap]{s} cm_2$ and $cm_1 \xrightarrow[vm, tmap]{ss'} cm_2$, where $s$ is a statement and $ss'$ is a sequence of statements. Note that $cm_1 \xrightarrow[tm, tmap]{s} cm_2$ and $cm_1 \xrightarrow[tm, tmap]{ss'} cm_2$ are mutually recursive, since a \prettyll{when} statement involves statement sequences. 
Finally, let $cm'$ be the connection mapping satisfying ${\sf emp} \xrightarrow[tm, tmap]{ss} cm'$, where ${\sf emp}$ is the empty connection mapping. The reason why $tm$ is still needed in the definition of the two relations lies in the fact that if $ss$ contains a statement of the form \prettyll{reg x: t clk with reset => (rsig, rval)}, then we should check the reset values in $tm(id(x), i)$'s are compatible with $rval$ (see the rule ``register'' in Figure~\ref{fig:firrtl-sem}).

In the sequel, we show how to define the relations $cm_1 \xrightarrow[tm, tmap]{s} cm_2$ and $cm_1 \xrightarrow[tm, tmap]{ss'} cm_2$. Similar to the construction of $tmap$, when defining the two relations, we should deal with the mismatch between the abstraction levels of connection mappings and $M$. 
%
Furthermore, we should take care of the flipping status of fields when defining the semantics of connection statements of the form \prettyll{r1 <= r2}, where \prettyll{r1, r2} are both references. For instance, in the following {\hifirrtl} program, the connect statement \prettyll{a <= b} will be split into the following two connections between ground-type components, \prettyll{b.f3 <= a.f1} and \prettyll{a.f2 <= b.f4}. Note that in \prettyll{b.f3 <= a.f1}, the connection flow is reversed since \prettyll{f1} is a flipped field. 
%

The relations $cm_1 \xrightarrow[tm, tmap]{s} cm_2$ and $cm_1 \xrightarrow[tm, tmap]{ss'} cm_2$ are defined by the rules in Fig.~\ref{fig:firrtl-sem}. The rule for the \prettyll{skip} statement is omitted since it does not modify $cm_1$, that is,  $cm_1 \xrightarrow[tm, tmap]{\mbox{skip}} cm_1$.
The rules in Fig.~\ref{fig:firrtl-sem} utilize the following functions and relations,
\begin{itemize} 
\item $\updatecm(cm_1, (v, m), ft, ls)$ modifies $cm_1(v, m), \cdots, cm_1(v, m+n-1)$ to $ls[0], \cdots, ls[n-1]$ respectively, where $n = \sizeofftype(ft)$,
%
\item $\updatecmref(cm_1, (v, m), ft, ls, (v', m'), ft', ls')$ updates $cm_1$ into $cm_2$ such that for each $0 \le i < n$, $cm_2(v, m+i) = ls'[i]$ if the $i$-th field in $ft$ is not flipped and $cm_2(v', m'+i) = ls[i]$ otherwise, where $n = \sizeofftype(ft)$, 
%
\item $\listgexp(e, ft)$ returns a list of ground-type expressions obtained by expanding $e$ according to $ft$, 
%
\item $\typecompatible(ft, ft', flipped)$ checks whether the two types $ft$ and $ft'$ are compatible and distinguish between whether $flipped = \ltrue$ or not, 
%
\item $\typeofref(r, tmap)$ returns the type of the reference $r$, 
%
\item $\invalidateref(cm_1, (v, m), ft, ori)$ may set $cm_1(v, m+i)$ to $\svinvalid$ for $i: 0 \le i < n$, depending on the orientation $ori$ and the flipping status of the $i$-th ground type in $ft$, where $ft$ is the type of some reference denoted by $(v, m)$ and $n = \sizeofftype(ft)$. 
\end{itemize}
The functions $\updatecm$, $\updatecmref$, $\invalidateref$ will be defined in Fig.~\ref{tab:update-vm-ct} and the functions $\listgexp$, $\typecompatible$, $\typeofref$ will be defined in Fig.~\ref{fig:list-gexp}-\ref{fig:type-ref}. 

Let us explain the rules of Fig.~\ref{fig:firrtl-sem}. 
\begin{itemize}
\item The ``wire'' rule updates each of $cm_1(v, 0), \cdots, cm_1(v, sz-1)$ to $\svinvalid$, where $s = \mbox{wire }x: t$,  $v$ is the identifier of $x$, $tmap(v) = (ft, ori)$, and $sz= \sizeofftype(ft)$.  
%
\item The ``node'' rule is similar to the ``wire'' rule, except that $cm_1(v, 0), \cdots, cm_1(v, sz-1)$ are updated to $ls[0], \cdots, ls[sz-1]$ respectively, where $s = \mbox{node }x = e$, $v$ is the identifier of $x$, $\typeofexp(e) = ft$, $sz= \sizeofftype(ft)$, and $ls = \listgexp(e, ft)$. 
%
\item The ``register'' rule first checks that $ft$ and $rft$ are compatible, moreover, $\listgexp(rval, rft)$ and $(tm(v,0), \cdots, tm(v, sz-1))$ are equivalent, then it updates $cm_1(v, 0), \cdots, cm_1(v, sz-1)$ to $ls[0], \cdots, ls[sz-1]$, where $s = \mbox{reg } x : t\ e\ \mbox{with: reset=>}  (rsig, rval)$, $id(x) = v$, $tmap(v) = (ft, ori)$, $rft = \typeofexp(rval, tmap)$, $sz = \sizeofftype(ft)$, and $ls = \listgexp(x, ft)$.
%
\item The ``connect:exp'' rule checks that $ori$, the orientation of $r$, is $\duplex$ or $\sink$,  then checks the compatibility of $rft$ and $eft$, moreover, it updates $cm_1(v, m), \cdots, cm_1(v, m+sz-1)$ to $ls[0], \cdots, ls[sz-1]$, where $s = r <= e$ such that $e$ is not a reference, $id(r) = (v, m)$, $(rft, ori) = \typeofref(r, tmap)$, $\typeofexp(e, tmap) = eft$, $sz = \sizeofftype(rft)$, and $ls = \listgexp(e, eft)$. 
%
\item The ``connect:ref'' rule checks that $ori$, the orientation of $r$, is $\duplex$ or $\sink$, and $ori'$, the orientation of $r'$, is $\duplex$ or $\source$, then checks the compatibility of $ft$ and $ft'$, moreover, it updates $cm_1$ into $cm_2$ by calling $\updatecmref(cm, (v, m), ft, ls, (v', m'), ft', ls')$, where $s = r <= r'$, $id(r)=(v,m)$, $id(r') = (v', m')$, $(ft, ori) = \typeofref(r, tmap)$, $(ft', ori') = \typeofref(r', tmap)$, $\listgexp(r, ft) = ls$, and $\listgexp(r', ft') = ls'$. 
%
\item The ``invalid'' rule calls $\invalidateref(cm_1, (v, m), ft, ori)$ to update $cm_1$, where $s = r \mbox{ is invalid}$, $id(r) = (v, m)$, and $(ft, ori) = \typeofref(r, tmap)$. 
%
\item The ``when'' rule first checks the type of $cond$ is $(\gtype, (\uint, 1))$, then computes $cmt$ from $cm_1$ by applying $sst$, and computes $cmf$ from $cm_1$ by applying $ssf$, finally let $cm_2 = \mux(cond, cmt, cmf)$, where $s  = \mbox{when } cond:\ sst \mbox{ else}:\ ssf$ and the $\mux$ function combines $cmt$ and $cmf$ pointwise into $cm_2$ by using the multiplexer operator $\mux(cond, \_, \_)$. 
\end{itemize}

Let us explain the rules for $\updatecm$, $\updatecmref$, and $\invalidateref$ in Fig.~\ref{tab:update-vm-ct}.
\begin{itemize}
\item The function $\updatecm(cm_1, id, ft, ls)$ updates $cm_1$ into $cm_2$ by modifying $cm_1(v, m+n)$ to $ls[n]$ for each $0 \le n < sz$, where $id = (v, m)$, $sz = \sizeofftype(ft)$. 
%
\item The function $\updatecmref(cm_1, id, ft, ls, id', ft', ls')$ first calls $lgt = \listgtyperef(ft, false)$ to compute a list $lgt$ from $ft$ (the type of $r$), where each element in $lgt$ is a pair $(gt, flipped)$ such that $gt \in \gtypes$ and $flipped$ is a Boolean value denoting whether the corresponding field is flipped or not. Then for each $n: 0 \le n < sz$, it updates $cm_1(v, m+n)$ to $ls'[n]$ if $lgt[n] = (\_, \lfalse)$, and updates $cm_1(v', m'+n)$ to $ls[n]$ if $lg[n] = (\_, \ltrue)$, where $id = (v, m)$ and $id' = (v', m')$. 
%
\item The function $\invalidateref(cm_1, id, ft, ori)$ first calls $lgt = \listgtyperef(ft, false)$ to compute a list $lgt$ from $ft$ (the type of $r$), where each element in $lgt$ is a pair $(gt, flipped)$ such that $gt \in \gtypes$ and $flipped$ is a Boolean value denoting whether the corresponding field is flipped or not. Then for each $n: 0 \le n < sz$, it updates $cm_1(v, m+n)$ to $\svinvalid$ if either ($lgt[n] = (\_, \lfalse)$ and $ori = \duplex$ or $\sink$), or ($lgt[n] = (\_, \ltrue)$ and $ori = \duplex$ or $\source$), where $id = (v, m)$. 
\end{itemize}

The function $\listgexp$ is defined in Fig.~\ref{fig:list-gexp}. 
Let us explain the rules $\listgexp: mux$ and $\listgexp: vector$. The explanation of the other rules are omitted for brevity. 
%
\begin{itemize}
\item In the rule $\listgexp: mux$, $e = \mux(c, e_1, e_2)$. Then $l1 = \listgexp(e1, ft)$ and $l2 = \listgexp(e2, ft)$ are computed. Moreover, we should check that the lengths of $l1$ and $l2$ are equal. Finally, the result $l$ is computed as $\mux(c, l1, l2)$, where the operator $\mux(c, \_, \_)$ is applied pointwisely to the elements in $l1$ and $l2$.

\item In the rule $\listgexp: vector$, $e = r[n]$. Then we compute $(ft', ori) = \typeofref(r)$ and $sz  = \sizeofftype(ft)$. The result $l$ is defined as $(l'[0])[n]\ ++\ \cdots ++\ (l'[sz-1])[n]$, where $l' = \listgexp(r, ft')$. 
\end{itemize}

The relations $\typecompatible(ft, ft', flipped)$ and  $\typecompatiblefields(ff, ff', flipped)$ are defined in Fig.~\ref{fig:type-compatible}. Let us explain the rules for them. 
\begin{itemize}
\item In the rule ``exp-gt'', $ft$ and $ft'$ are explicit ground types, that is, types from $\{\uint, \sint\} \times \Nat \cup \{\clock, \reset, \asyncreset\}$, and required to be equal. 
%
\item In the rule ``imp-gt-1'',  $ft = (\gtype, t)$, $ft' = (\gtype, t')$, $t$ is implicit and $t'$ is explicit, that is, either $t= (\uintimp, n)$ and $t' = (\uint, n')$, or $t= (\sintimp, n)$ and $t' = (\sint, n')$.  If $flipped = \lfalse$, then the connection flow is from $ft'$ to $ft$, thus it is required that $n \ge n'$. 
%
\item In the rule ``imp-gt-2'',  $ft = (\gtype, t)$, $ft' = (\gtype, t')$, $t$ is explicit and $t'$ is implicit, that is, either $t= (\uint, n)$ and $t' = (\uintimp, n')$, or $t= (\sint, n)$ and $t' = (\sintimp, n')$.  If $flipped = \ltrue$, then the connection flow is from $ft$ to $ft'$, thus it is required $n \le n'$. 
%
\item In the rule ``imp-gt-3'',  $ft = (\gtype, t)$, $ft' = (\gtype, t')$, both $t$ and $t'$ are implicit, that is, $t= (\uintimp, n)$ and $t' = (\uintimp, n')$, or $t= (\sintimp, n)$ and $t' = (\sintimp, n')$.  If $flipped = \lfalse$, then $n \ge n'$, and $n \le n'$ otherwise. 
%
\item In the rule ``vector'', $ft = (\vtype, t[n])$ and $ft' = (\vtype, t'[n])$, then it is required that $t$ and $t'$ is compatible, that is, $\typecompatible(t, t', flipped)$ holds. 
%
\item In the rule ``bundle'', $ft = (\btype, ff)$ and $ft' = (\btype, ff')$. Then it is required that $\typecompatiblefields(ff, ff', flipped)$ holds. 
%
\item In the rule ``nil'', $\typecompatiblefields(ff, ff', flipped)$ holds iff $ff = ff' = \nil$. 
%
\item In the rule ``$\neq$nil-false'', $ff = (f, b, t)\ ff1$, $ff' = (f', b, t')\ ff1'$, and $b = \lfalse$. Then it is required that $\typecompatible(t, t', flipped)$ and $\typecompatiblefields(ff1, ff'1, flipped)$ hold. 
%
\item In the rule ``$\neq$nil-true'', $ff = (f, b, t)\ ff1$, $ff' = (f', b, t')\ ff1'$, and $b = \ltrue$. Then it is required that $\typecompatible(t, t', \neg flipped)$ and $\typecompatiblefields(ff1, ff'1, flipped)$  hold. Note that because $b = \ltrue$, that is, $f$ and $f'$ are flipped fields, the value $\neg flipped$, instead of $flipped$, is used in $\typecompatible(t, t', \neg flipped)$.
