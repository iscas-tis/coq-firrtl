The function $\madeexplicit(ft, id, tm)$ is defined  in Fig.~\ref{fig:made-explicit}. 
\begin{itemize}
\item The rule {\madeexplicit: \gtype} deals with the case $ft = (\gtype, gt)$, which  simply returns the ground type in $tm(id)$. 
%
\item The rule {\madeexplicit: \vtype} deals with the case $ft = (\vtype, t[n])$. In this case, $\madeexplicit$  computes $t' = \madeexplicit(t, id, tm)$ and  returns $t'[n]$. Moreover, it checks that the $n$ ground-type sequences in $tm$ corresponding to the $n$ copies of $t'$ are all equal, that is, let $id = (v,m)$ and $sz = \sizeofftype(t')$, then $(tm(v, m+0), \ldots, tm(v, m+sz-1)) = (tm(v, m+sz), \ldots, tm(v, m+2sz-1)) = \cdots = (tm(v, m+(n-1)sz), \ldots, tm(v, m+nsz-1))$. \tl{to see how to improve this.}
%
\item The rule {\madeexplicit: \btype} deals with the case $ft = (\btype, \vec{f})$. In this case, $\madeexplicit$ calls $\madeexplicitfields(\vec{f}, id, tm)$ and returns its output. $\madeexplicitfields(\vec{f}, id, tm)$ %recursively computes the output as follows. 
processes the fields sequentially. 
For the field $(x,b,t)$, (recursive) call to $\madeexplicit(t, id, tm)$ returns $t'$ and thus we obtain $(x,b,t')$. 
%If $\vec{f} = \nil$, then it returns $(\btype, \nil)$. Otherwise, let $\vec{f} = (x, b, t)\ \myvec{f'}$. Then it calls $\madeexplicit(t, id, tm)$ to get $t'$ and recursively calls $\madeexplicitfields(\myvec{f'}, id, tm)$ to get $\myvec{f''}$. Finally, it returns $newt = (x, b, t')\ \myvec{f''}$. 

\end{itemize}

For instance, in the rule {\madeexplicit: \vtype}, if the $n$ ground-type sequences in $tm$ corresponding to the $n$ copies of $t'$ are not equal, then $\madeexplicit$ returns the special value $\svnone$.

The mapping $tmap$ is defined by the procedure $\portsstmtstmap(pp, ss, tm)$, which calls the procedure $\portstmap(pp, tm)$ to compute $pmap$, then calls $\stmtstmap((pmap, pmap), ss, tm)$ \tl{to check} to get $tmap$ (see Figure~\ref{fig:ports-stmts-tmap}). 
%
Intuitively, $\portstmap(pp, tm)$ defines a mapping based on the port declarations $pp$ and $tm$, and $\stmtstmap((pmap, pmap), ss, tm)$ defines a pair $(tmap, scope)$ from $(pmap, pmap)$, the statement sequence $ss$, and $tm$.

Note that $scope$ is introduced to check that the component references in $ss$ do not violate the scoping constraints brought upon by the \prettyll{when} statements. For instance, in the following {\hifirrtl} program, there is a scoping error, since $z$ is declared in the scope of the \prettyll{when} statement, but it is referred outside it (Line 7). 
%
%The procedures $\portstmap$ and $\stmtstmap$ are defined by the rules in Fig.~\ref{fig:ports-stmts-tmap}.

$\portstmap(pp, tm)$ defines $tmap$ from the port declarations in $pp$. If $pp = input\ x: t\ pp'$, then it calls $\madeexplicit(t, (id(x), 0), tm)$ to construct $newt$. Moreover, it recursively calls $\portstmap(pp', tm)$ to obtain $tmap'$. Then it checks that $tmap'(id(x)) = \svnone$.  If it is the case, then the output $tmap$ is obtained by adding the pair $(id(x), (newt, \source))$ to $tmap'$. The case $pp = output\ x: t\ pp'$ is dealt with similarly, with $\source$ replaced by $\sink$. Furthermore, if $pp = \nil$, then the empty map $\sf emp$ is returned. 

$\stmtstmap((tmap, scope), ss, tm)$ defines $tmap$ from the statements in $ss$. If $ss = \nil$, then it simply returns $(tmap, scope)$. Otherwise, let $ss = s\ ss'$, where $s$ is a statement. Then it calls $\stmttmap(s, tm)$ to compute $(tmap'', scope'')$ from a $s$ (see Figure~\ref{fig:stmts-tmap} for its definition). Finally, it recursively calls $\stmtstmap((tmap'', scope''), ss', tm)$ to compute the return value $(tmap', scope')$.

The procedure $\stmttmap$ is defined in Fig.~\ref{fig:stmts-tmap}, the intuitions of which is as follows. 

If $s$ is a declaration statement, then $\stmttmap((tmap, scope), s, tm)$ utilizes $tm$ to instantiate the types in $s$ and update $(tmap, scope)$ into $(tmap', scope')$. We distinguish between different cases of declaration statements. 
\begin{itemize}
	\item If $s = \mbox{wire } x: t$, then it checks $tmap(id(x)) = \svnone$ and calls $\madeexplicit(t, (id(x), 0), tm)$ to compute $newt$. Finally, it adds the pair $(id(x), (newt, \duplex))$ to both $tmap$ and $scope$, and obtains the return value $(tmap', scope')$. Note that the orientation of a \prettyll{wire} component is \duplex. (see the rule \stmttmap: wire in Fig.~\ref{fig:stmts-tmap})
	%
	\item If $s = \mbox{node } x = e$, then it checks $tmap(id(x)) = \svnone$, computes $t= \typeofexp(e, scope)$, and calls $\madeexplicit(t, (id(x), 0), tm)$ to obtain $newt$. Finally, it adds $(id(x), (newt, \source))$ to both $tmap$ and $scope$, and obtains the return value $(tmap', scope')$. Note that in the function call $\typeofexp(e, scope)$, $scope$, instead of $tmap$, is used to check whether the references in $e$ are well-scoped. Moreover, the orientation of a \prettyll{node} component is \source. (see the rule \stmttmap: node in Fig.~\ref{fig:stmts-tmap})
	%
	\item If $s = \mbox{reg } x: t \ clk \mbox{ with: reset =>} (rsig, rval)$ and additionally $\typeofexp(rsig, scope) = (\gtype, \asyncreset)$, then it checks $tmap(id(x)) = \svnone$, $\typeofexp(clk, scope) \neq \svnone$, and $\typeofexp(rval, scope) \neq \svnone$, calls $\madeexplicit(t, (id(x), 0), tm)$ to obtain $newt$. Finally, it adds $(id(x), (newt, \duplex))$ to both $tmap$ and $scope$, and obtains the return value $(tmap', scope')$. Note that in $\typeofexp(clk, scope)$, $\typeofexp(rsig, scope)$, and $\typeofexp(rval, scope)$, $scope$, instead of $tmap$, is used to check that the references in $clk, rsig, rval$ are well-scoped. Moreover, the orientation of a \prettyll{reg} component is \duplex. (see the rule \stmttmap: reg-asyncreset in Fig.~\ref{fig:stmts-tmap})
	%
	\item If $s = \mbox{reg } x: t \ clk \mbox{ with: reset =>} (rsig, rval)$ and additionally $\typeofexp(rsig, scope) = (\gtype, (\uint, 1))$, then it checks $tmap(id(x)) = \svnone$ and $\typeofexp(clk, scope) \neq \svnone$, calls $\madeexplicit(t, (id(x), 0), tm)$ to obtain $newt$. Moreover, it adds $(id(x), (newt, \duplex))$ to $scope$ and obtains $scope'$. Then it checks $\typeofexp(rval, scope') \neq \svnone$, and adds $(id(x), (newt, \duplex))$ to $tmap$ and obtains $tmap'$. Finally, it returns $(tmap', scope')$. Note that in $\typeofexp(clk, scope)$, $\typeofexp(rsig, scope)$, $scope$, instead of $tmap$, is used to check that the references in $clk, rsig$ are well-scoped. Moreover, because the reset signal $rsig$ is synchronous, that is, $\typeofexp(rsig, scope) = (\gtype, (\uint, 1))$, we have that in $\typeofexp(rval, scope')$, $scope'$, instead of $scope$ or $tmap$, is used to check that the references in $rval$ are well-scoped. (see the rule \stmttmap: reg-reset in Fig.~\ref{fig:stmts-tmap})
\end{itemize}

For others, if $s$ is a connect or invalid statement,  $\stmttmap$ %((tmap, scope), s, tm)$ 
only checks that the references or expressions are well-scoped with respect to $scope$, but does not modify $(tmap, scope)$. %(see the rules \stmttmap: connect and  \stmttmap: invalid in Fig.~\ref{fig:stmts-tmap})
%

If $s$ is a when statement \prettyll{when c: sst else: ssf},  $\stmttmap$ %((tmap, scope), s, tm)$ 
first calls $\stmtstmap((tmap, scope), sst, tm)$ to compute $(tmapt, scopet)$, then calls  $\stmtstmap((tmapt, scope), ssf, tm)$ to compute $(tmapf, scopef)$, finally returns the pair $(tmapf, scope)$. Note that in $\stmtstmap((tmapt, scope), ssf, tm)$, we are using $scope$, instead of $scopet$, since the components declared in $sst$ should \emph{not} be referred in $ssf$.  Moreover, the return value of $\stmttmap((tmap, scope), s, tm)$ is $(tmapf, scope)$, instead of $(tmapf, scopef)$, since the components declared inside the when statement should \emph{not} be referred outside it. 
%(see the rule \stmttmap: when in Fig.~\ref{fig:stmts-tmap})
