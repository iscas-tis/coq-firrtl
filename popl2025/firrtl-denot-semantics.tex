Since each component $cnt$ in $M$ is declared, $cnt$ has a type, say $type(cnt)$. Note that $type(cnt)$ may involve implicit widths.  
On the other hand, all the widths $vm$ are explicit. We can construct a function $tmap$ that assigns to each identifier $v \in \range{id}$ a type where all the widths are explicit, by utilizing the information in $vm$ to instantiate the implicit widths in $type(cnt)$ for components $cnt$ in $M$. More precisely, for each $v \in \range{id}$, $tmap(v)$ is of one of the following forms, 
\begin{itemize}
\item $(ground, t)$ such that $t \in \gtypes$, 
%
\item $(vector, t[n])$ such that $t$ is a type where all widths are explicit, 
%
\item $(bundle, f)$ such that $f$ is a field where all widths are explicit. 
\end{itemize}

In the sequel, we first assume that $tmap$ is already constructed and show how to define the semantics of $M$. Later on, we show how to construct $tmap$ from $M$ and $vm$.

Moreover, the relation can be easily extended to sequences of statements, that is, $(vm, ct) \xrightarrow[tmap]{ss} (vm', ct')$, where $ss$ is a sequence of statements.

Fixpoint size_of_ftype ft :=
  match ft with
  | Gtyp t => 1
  | Atyp t n => (size_of_ftype t) * n
  | Btyp b => size_of_fields b
  end
with size_of_fields b :=
       match b with
       | Fnil => 0
       | Fflips v fl t fs => (size_of_ftype t) + size_of_fields fs
       end.
