In this section, we define the formal semantics of {\hifirrtl} programs. Since a {\hifirrtl} program provides a high-level description of low-level RTL circuits, %the main idea of 
the formal semantics is essentially to specify which low-level RTL circuits conform to a given {\hifirrtl} program. 

Technically, a {\hifirrtl} program is %\emph{nondeterministic} 
a under-specification in the sense that bit widths of some of its components are not determined. As a result, the formal semantics of a given {\hifirrtl} program is defined as the set of (low-level) {\lofirrtl} programs that conform to it, where all components are of ground types, whose bit widths are all explicit given, and each component is connected at most once. 

We shall represent {\lofirrtl} programs %by a data structure called \emph{module graphs}. A module graph $G = (vm, ct)$ represents a module in {\hifirrtl} programs, 
as a pair $(vm, ct)$ where $vm$ records the typing information of ground-type components in the module and $ct$ records the connections between these components.  
%
A {\hifirrtl} module %$M$, \emph{the semantics of $M$ is defined as 
is interpreted as the set of such pairs that conform to it. 


Moreover, a module graph $G= (vm, ct)$ \emph{conforms to} a {\hifirrtl} module $M$ is defined as follows: \tl{to see how to improve this.}
\begin{itemize}
	\item At first, $vm$ is used to determine the implicit bit widths in $M$ and build a type mapping $tmap$, which records the typing information for all the components in $M$, so that all the widths in it are explicit. (Note that the components of $M$ may not be of ground types. ) 
	%
	\item Then, to decide whether $(vm, ct)$ conforms to $M$, we utilize $tmap$ and the statements in $M$ to construct a module graph $G'$ from the empty module graph and finally check whether $G' = G$.
\end{itemize}
